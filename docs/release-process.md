# MiniMem Release Process

## Pre-Release Checklist

### 1. Build Verification
- [ ] Userspace library builds: `meson setup build && meson compile -C build`
- [ ] Kernel module builds: `./build-kmod.sh build`
- [ ] No compiler warnings (except pre-existing WKdm frame-size)
- [ ] DKMS packaging tests pass: `./tests/test_dkms_packaging.sh`

### 2. Test Verification
- [ ] All Criterion unit tests pass (if criterion installed): `meson test -C build`
- [ ] Kernel module kselftest passes in QEMU VM: `./vm-test-minimem.sh`
- [ ] E2E test suite passes in QEMU VM: `./tests/kernel/minimem_e2e_test.sh`
- [ ] Transparent E2E test passes in QEMU VM: `test_transparent_e2e`
- [ ] DKMS packaging test: 48/48 checks pass

### 3. Documentation
- [ ] `docs/feature-registry.md` is up to date
- [ ] `docs/roadmap.md` reflects current stage
- [ ] `NearTermTodos.txt` has no stale completed items
- [ ] `CONTRIBUTING.md` reflects current contribution layers
- [ ] All research docs referenced from design docs

### 4. Version Bump
- [ ] Update version in `meson.build` (userspace lib version)
- [ ] Update version in `dkms/dkms.conf` (PACKAGE_VERSION)
- [ ] Update version in `dkms/install.sh` and `dkms/uninstall.sh` (path)
- [ ] Update version in `scripts/dkms-install.sh` and `scripts/dkms-uninstall.sh`
- [ ] Update version in `packaging/aur/minimem/PKGBUILD` (pkgver)
- [ ] Update version in `packaging/aur/minimem-dkms/PKGBUILD` (pkgver)
- [ ] Update version in `packaging/aur/minimem-dkms/minimem-dkms.install`
- [ ] Update version in `packaging/debian/control` and `packaging/debian/rules`
- [ ] Update version in `packaging/fedora/minimem.spec`
- [ ] Update version in `src/kernel/minimem_main.c` (MODULE_VERSION)
- [ ] Verify all versions are consistent: `grep -r '0\.6\.0'`

### 5. Git Tag
- [ ] Commit all changes
- [ ] Tag: `git tag -s v0.6.0 -m "MiniMem v0.6.0"`
- [ ] Push tag: `git push origin v0.6.0`

### 6. GitHub Release
- [ ] Create release from tag on GitHub
- [ ] Include benchmark summary in release notes
- [ ] Attach source tarball (auto-generated by GitHub)

### 7. AUR Submission
- [ ] Generate `.SRCINFO`: `makepkg --printsrcinfo > .SRCINFO`
- [ ] Clone AUR repo: `git clone aur@aur.archlinux.org:minimem.git`
- [ ] Copy PKGBUILD + .SRCINFO + .install to AUR repo
- [ ] Commit + push to AUR
- [ ] Verify: `yay -S minimem` and `yay -S minimem-dkms` work on clean system
- [ ] Repeat for `minimem-dkms` AUR package

### 8. Post-Release
- [ ] Announce on GitHub Discussions
- [ ] Update `docs/feature-registry.md` with release date
- [ ] Create next milestone issues on GitHub

## Version Policy

- **Semantic versioning**: MAJOR.MINOR.PATCH
- **MAJOR**: Architecture change (new compression paradigm, kernel API break)
- **MINOR**: New features (new algorithm, new sysfs attribute, new test)
- **PATCH**: Bug fixes (crash fix, build fix, documentation fix)
- **Kernel module version**: Set separately via `MODULE_VERSION()` in `minimem_main.c`
- **Userspace library version**: Set in `meson.build` with soname versioning

## AUR Maintenance

- Update PKGBUILD `pkgver` and `sha256sums` on each release
- GPG-sign commits to AUR repo
- Monitor AUR comments for build failures
- Test on clean Arch install before pushing updates
